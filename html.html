HTML 01. p태그가 br태그보다 좋은 점
-> br 태그는 단순히 줄바꿈을 의미할뿐이지만 p태그는 줄바꿈뿐만 아니라, 단락을 구분하는 태그이다. p태그를 사용했을 때 단락을 구분하는 형식이 자유롭지 않으나 css와 함께 사용한다면 그 단점을 극복할 수 있다.

HTML 02. ul태그와 ol태그의 차이점
-> ul태그는 목차를 정리해주지만 넘버링을 안해주고, ol태그는 넘버링을 해준다.

HTML 03. 한글을 출력하고자할 때는 meta 태그를 이용하여 UTF-8를 지정해줘야합니다.
-> meta charset="utf-8"과 같은 형태로 사용합니다.

HTML 04. HTML 문서를 제작할 때의 지켜야하는 문서의 구조
->반드시 html, head, body 태그 를 사용하여 제작해야합니다. 이 태그들을 안 사용한다고 웹 페이지가 구축되지 않는 것은 아니지만 유의미한 정보를 담은 웹 페이지라면 소스를 열어보았을 때 문서의 구조가 정확히 보이도록 코딩해야히기 때문입니다.

HTML 05. 링크를 표현하는 a태그
-> a태그는 링크를 표현하는 태그이며, HTML 태그의 제왕이라고 불립니다.
